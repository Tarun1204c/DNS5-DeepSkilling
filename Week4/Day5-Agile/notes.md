# Week4 Day5 - Agile Methodology (Module 14)

## What is Agile?
A project management approach that breaks work into small, iterative cycles 
(sprints) instead of delivering everything at once (like in Waterfall model).

## Agile vs Waterfall

### Waterfall
- Sequential: Requirements → Design → Build → Test → Deploy (one after another)
- Changes are hard to make once a phase is done
- Works well for fixed, well-defined projects

### Agile
- Iterative: Work is done in small cycles (sprints), typically 1-4 weeks
- Continuous feedback and adaptation
- Works well for projects that evolve over time

## Key Agile Concepts

### Sprint
A fixed time period (usually 2 weeks) where a specific set of tasks is completed.

### Scrum
The most common Agile framework, with defined roles and ceremonies.

### Scrum Roles
- Product Owner - defines what needs to be built (priorities, requirements)
- Scrum Master - facilitates the process, removes blockers for the team
- Development Team - builds the actual product

### Scrum Ceremonies (Meetings)
- Sprint Planning - deciding what to work on in the upcoming sprint
- Daily Standup - short daily meeting (what I did, what I'll do, blockers)
- Sprint Review - demo completed work to stakeholders
- Sprint Retrospective - team discusses what went well/what to improve

### Backlog
A prioritized list of tasks/features to be worked on (Product Backlog = 
overall list, Sprint Backlog = tasks for current sprint).

### User Story
A feature described from the user's perspective, e.g.:
"As a student, I want to view my course details, so that I can track my progress."

## Tools Used in Agile
- Jira - task tracking and sprint management
- Trello - simpler kanban-style board
- Azure DevOps Boards - similar to Jira, used in Microsoft ecosystem

## Why It Matters
Most companies (including Cognizant projects) use Agile/Scrum for development. 
Understanding sprints, standups, and backlogs is essential for working in 
any real development team.
