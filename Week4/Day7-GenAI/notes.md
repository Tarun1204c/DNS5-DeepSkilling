# Week4 Day7 - Generative AI Basics (Module 16)

## What is Generative AI?
AI systems that can create new content (text, images, code, audio) based 
on patterns learned from large amounts of training data, rather than just 
analyzing or classifying existing data.

## Key Concepts

### LLM (Large Language Model)
AI models trained on massive text datasets to understand and generate 
human-like text. Examples: GPT (OpenAI), Claude (Anthropic), Gemini (Google).

### Prompt Engineering
The practice of crafting effective inputs (prompts) to get better, more 
accurate outputs from AI models.

### Generative AI Use Cases in Software Development
- Code generation and code completion (e.g., GitHub Copilot)
- Automated documentation writing
- Bug detection and code review assistance
- Test case generation
- Chatbots and customer support automation

## GenAI in Enterprise (Cognizant context)
- Used to accelerate development (like this DN5.0 program itself, using 
  AI-assisted learning)
- Automating repetitive coding tasks
- AI-powered code reviews and quality checks
- Building AI-integrated applications (chatbots, recommendation systems)

## Popular GenAI Tools
- ChatGPT / Claude - conversational AI assistants
- GitHub Copilot - AI pair programmer for code
- Midjourney / DALL-E - image generation

## Ethical Considerations
- Data privacy (what data is used to train/fine-tune models)
- Bias in AI-generated outputs
- Over-reliance on AI without understanding underlying logic

## Why It Matters
GenAI is increasingly integrated into development workflows (code assistants, 
automated testing, documentation). Understanding its capabilities and 
limitations is valuable for any modern developer.
